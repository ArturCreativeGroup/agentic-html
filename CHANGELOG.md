# v0.2.0-experimental
## Operational Semantics Release

Fecha:
2026

Estado:
Experimental

---

# Resumen

AHP v0.2 introduce una capa semántica operacional orientada a modificar comportamiento de agentes autónomos durante interacción web real.

Esta versión amplía el enfoque estructural introducido en v0.1 incorporando:

- semántica operacional
- políticas de ejecución
- navegación determinista
- reducción heurística
- movement batching
- dangerous action awareness
- reanalysis policy
- benchmark framework experimental

---

# Nuevas Capacidades

## AHP-SKILL.md

Nueva especificación operacional para agentes autónomos.

Incluye:
- navegación determinista
- targeting semántico
- continuidad operacional
- dangerous action avoidance
- grouped execution
- reanalysis policy
- scroll behavior guidance
- interaction targeting

---

## Semántica Operacional

AHP evoluciona desde:
- semántica estructural

hacia:
- semántica operacional

El protocolo ahora describe:
- cómo navegar
- cómo interactuar
- cómo reducir ambigüedad
- cómo evitar errores
- cómo mantener continuidad operacional

---

## Dangerous Action Awareness

Nueva filosofía de seguridad operacional.

Incluye:
- evitar Enter prematuro
- evitar submits implícitos
- evitar acciones irreversibles
- confirmación explícita
- targeting controlado

---

## Grouped Execution

Nueva guía operacional para:
- batching de acciones
- reducción de context switching
- menor fragmentación operacional
- menor densidad de pausas cognitivas

Ejemplos:
- click + type + blur
- open + inspect + summarize

---

## Reanalysis Policy

Nueva política de reinterpretación operacional.

Principios:
- reanalizar solo ante fallo
- reinterpretación localizada
- evitar reconstrucción completa de interfaz
- preservar continuidad operacional

---

# Resultados Experimentales Iniciales

## ObraLink Test

Resultados observados:
- ~18.2% reducción estimada de tiempo
- ~20% reducción estimada de pasos

Cambios observados:
- menor exploración heurística
- mayor claridad operacional
- mejor navegación semántica

---

## Qwen Desktop Test

Resultados observados:
- ~37.1% reducción estimada de pasos

Cambios observados:
- menor ambigüedad operacional
- mejor targeting
- navegación más directa

---

## ObraLink AHP v0.2

Resultados observados:
- 100% task completion
- 0 errores fatales
- 2 scrolls
- 4 reintentos menores

Cambios operacionales observados:
- batching de movimientos
- menor pausa cognitiva
- mejor continuidad operacional
- reanálisis localizado
- avoidance de Enter

- accesibilidad web
- navegación humana

- APIs propietarias
- runtimes especiales
- browsers modificados

# Limitaciones de v0.1

v0.1 aún dependía fuertemente de:
- exploración heurística
- navegación reactiva
- targeting ambiguo
- reinterpretación frecuente

Estas limitaciones motivaron:
- AHP-SKILL.md
- semántica operacional
- benchmark framework
- deterministic interaction guidance

---

# Principio General

AHP es un protocolo experimental.

El objetivo del proyecto no es reemplazar la web humana.

El objetivo es investigar cómo interfaces web pueden exponer mejor contexto operacional para agentes inteligentes.
