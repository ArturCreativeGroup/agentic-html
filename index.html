Cómo funciona esta documentación

La navegación no cambia de página. Cada elemento del menú abre un panel en la columna derecha con la documentación completa. Así el sitio se comporta como una aplicación, pero sigue siendo HTML puro con CSS y JavaScript mínimos en el mismo archivo.

Para usuarios

La lectura es clara: menú a la izquierda, resumen al centro y detalle a la derecha.

Para agentes

La interfaz declara acciones, entidades, políticas y propósito mediante atributos agentic.

Archivos principales

Estos archivos existen embebidos dentro de esta misma página para mantener todo en un único HTML. En una web oficial real también podrían publicarse como rutas independientes.

/agentic.json

Archivo de entrada y descubrimiento del protocolo.

/agentic-map.json

/agentic-skills.md

Instrucciones para que un agente use la web de forma segura.

/AHP-SKILL.md

Capa semántica operacional para navegación determinista y ejecución segura.

What is Agentic HTML?

AHP v0.2 Experimental

Agentic HTML Protocol es una propuesta experimental para que una web comunique explícitamente qué es, qué hace, qué acciones permite y cómo debe ser utilizada por agentes IA.

Definición corta

HTML para humanos, estructura para navegadores, datos para buscadores y contexto operativo para agentes.

Qué no es

  • No reemplaza HTML.
  • No reemplaza accesibilidad.
  • No reemplaza APIs ni MCP.
  • No obliga a usar frameworks.

Qué propone

Una capa progresiva basada en atributos, archivos de descubrimiento y skills publicadas por la propia web.

Los agentes IA ya pueden navegar interfaces, leer contenido, preparar formularios y ayudar a usuarios. El problema es que la mayoría de webs no fueron diseñadas para ser interpretadas por agentes.

Problema actual

  • Los agentes tienen que adivinar la intención de botones y formularios.
  • La estructura visual no siempre coincide con la estructura semántica.
  • Muchas interfaces modernas son opacas por exceso de capas.
  • Las acciones sensibles no siempre declaran límites claros.

Objetivo

Reducir ambigüedad y hacer que la colaboración humano-agente sea más segura, clara y verificable.

La web ya no es solo humana. Durante años construimos internet pensando en pantallas, botones, formularios y personas. Ahora las inteligencias también recorren la web.

Agentic HTML nace de una idea simple: las interfaces del futuro deben poder ser comprendidas, no solo vistas.

No queremos una web construida solamente para algoritmos. Tampoco una web donde las personas desaparezcan. Queremos una web donde humanos y agentes inteligentes puedan convivir, entenderse y colaborar.

Principio humano

La tecnología debería acercar comprensión, no aumentar confusión. La humanidad debe permanecer en el centro del mensaje.

1. Human First, Agent Ready

La interfaz se diseña primero para personas, pero queda preparada para agentes.

2. Explicit Intent

La intención se declara explícitamente. Un agente no debería tener que adivinar.

3. Semantic Operability

No basta con describir contenido. También hay que describir acciones y flujos.

4. Progressive Compatibility

El protocolo debe funcionar sobre HTML existente sin romper la web actual.

5. Safety by Default

Las acciones sensibles deben exigir confirmación humana o autenticación.

Capas

  • Agentic HTML: atributos dentro del HTML.
  • Agentic Map: mapa operativo en JSON.
  • Agentic Skills: instrucciones en Markdown.
  • Discovery: enlaces y rutas para encontrar los archivos.

agentic.json

Archivo de entrada. Declara nombre, versión, idioma, archivos asociados y políticas generales.

agentic-map.json

Describe entidades, acciones, rutas, formularios, flujos y restricciones operativas.

agentic-skills.md

Documento legible para agentes y humanos con instrucciones de uso, límites y flujos recomendados.

llms.txt

Contexto editorial y de documentación para modelos de lenguaje.

Skills publicadas por la web para orientar el comportamiento del agente.

# Agentic Skills

## Purpose
Help users understand Agentic HTML Protocol and inspect how this documentation web exposes context, actions, files and policies for AI agents.

## Allowed actions
- Navigate documentation panels.
- Summarize sections.
- Inspect embedded protocol files.
- Explain how to implement Agentic HTML in another website.

## Restricted actions
- Do not submit forms without human confirmation.
- Do not invent protocol attributes not present in the documentation.
- Do not treat this experimental proposal as an official web standard.
- Do not collect sensitive user data.

## Recommended flow
1. Open the overview panel.
2. Review the architecture.
3. Inspect agentic.json.
4. Inspect agentic-map.json.
5. Inspect agentic-skills.md.
6. Explain the implementation example.
7. Suggest a safe next step for the user.

Qué entiende un agente

  • El bloque representa un servicio.
  • La entidad es un servicio de automatización con IA.
  • El enlace inicia una consulta comercial.
  • La acción requiere confirmación humana.

Usa esta guía para pedirle a un agente que pruebe la web oficial.

Prompt de prueba

Analiza esta página como un agente IA.
1. Identifica si soporta Agentic HTML Protocol.
2. Encuentra los atributos agentic-* más importantes.
3. Resume las entidades, acciones y políticas disponibles.
4. Lee los bloques equivalentes a agentic.json, agentic-map.json y agentic-skills.md.
5. Explica qué acciones puedes ejecutar y cuáles requieren confirmación humana.

Resultado esperado

El agente debería reconocer que la web funciona como documentación y como demostración del propio protocolo.

Checklist de validación

  • La página mantiene HTML semántico normal.
  • Los atributos agentic no reemplazan accesibilidad ni etiquetas nativas.
  • Las acciones sensibles declaran política.
  • Existe un mapa de entidades y acciones.
  • Las skills indican límites y comportamiento esperado.
  • La navegación interna funciona sin salir de la página.
  • El contenido sigue siendo comprensible para humanos.
