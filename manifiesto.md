# Manifiesto — Agentic HTML Protocol

## Human First. Agent Ready.

La web cambió.

Durante años construimos interfaces pensadas principalmente para personas y navegadores.
Ahora comenzamos a entrar en una nueva etapa donde agentes IA también interactúan, interpretan y navegan sistemas digitales.

Pero las interfaces actuales todavía obligan a los agentes a:

* inferir intención
* interpretar estructuras ambiguas
* navegar visualmente
* adivinar workflows
* deducir contexto

La web moderna funciona.
Pero quizá todavía no comunica suficiente intención operacional.

---

## Este proyecto no busca reemplazar HTML.

Tampoco busca reemplazar la experiencia humana de la web.

Agentic HTML Protocol nace como una investigación experimental para explorar una pregunta:

> ¿Puede una interfaz comunicar su intención de forma más explícita para agentes IA?

La idea no es construir una web para máquinas.
La idea es explorar una web capaz de comunicar mejor.

---

## La interfaz como contexto operacional

Una interfaz no es solo diseño.

También contiene:

* acciones
* decisiones
* workflows
* restricciones
* prioridades
* relaciones
* contexto
* intención

Actualmente gran parte de esa información está implícita.

Agentic HTML explora la posibilidad de volver parte de ese contexto más explícito.

---

## Humanidad primero

La IA no debería eliminar claridad humana.

Una interfaz sigue existiendo primero para personas.

La intención de este proyecto no es transformar la web en una estructura fría para automatización.

La intención es investigar cómo humanos y agentes pueden compartir mejor el mismo espacio operacional.

Human First.
Agent Ready.

---

## Una investigación, no una promesa

Este proyecto es experimental.

No es:

* un estándar oficial
* un reemplazo de HTML
* una especificación definitiva
* una solución validada a gran escala

Actualmente es:

* una hipótesis
* una exploración técnica
* un laboratorio experimental
* una investigación abierta

---

## Menos ambigüedad

Muchos agentes actuales dependen de:

* heurísticas
* inferencias
* navegación probabilística
* interpretación visual

Eso funciona.
Pero también genera:

* errores
* inconsistencias
* consumo innecesario de contexto
* workflows poco claros

Agentic HTML investiga si una capa semántica-operacional podría reducir parte de esa ambigüedad.

---

## Interfaces comprensibles

La visión detrás del proyecto es simple:

> una interfaz podría comunicar no solo cómo se ve,
> sino también qué representa,
> qué acciones existen,
> qué restricciones tiene,
> y qué intención operacional posee.

---

## El protocolo como mapa

Agentic HTML no pretende reemplazar la interfaz real.

La interfaz sigue siendo el territorio.

El protocolo explora la idea de crear:

* mapas operacionales
* relaciones semánticas
* workflows explícitos
* políticas de interacción
* contexto estructurado

Una capa complementaria.
No un sustituto.

---

## Investigación abierta

El proyecto todavía tiene múltiples limitaciones:

* benchmarks reducidos
* inferencia experimental
* workflows inestables
* relationship graphs incompletos
* falta de validación multi-modelo

Y precisamente por eso existe públicamente.

Para:

* experimentar
* iterar
* aprender
* recibir críticas
* validar hipótesis
* construir evidencia real
